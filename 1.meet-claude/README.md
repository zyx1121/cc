# 第 1 章 — 認識 Claude Code:對話、動手、記憶

上一章你把 Claude Code 裝好、登入、也接上 GitHub 了。這一章先不急著做大東西,而是花點時間搞懂它三件最核心的事:**怎麼跟它對話、它怎麼幫你動手改檔、還有它怎麼「記住」你**。

到最後,你會親手撞見它在「記憶」上的一個小麻煩 —— 那個麻煩,正是下一章我們要動手做工具來解決的。

全程你只要打字跟它說話,指令它會幫你跑。

---

## 開始前:開一個練習資料夾

第 0 章你已經會開終端機了。這次我們在一個**新資料夾**裡啟動 Claude Code。先建一個資料夾(例如桌面上的 `cc-practice`),在它裡面開終端機,然後:

```
claude
```

> 為什麼要「在資料夾裡」啟動?因為 Claude Code 是以**資料夾(專案)**為單位工作的 —— 它把這個資料夾當成它的工作範圍,之後讀寫檔案、記憶,都跟這個資料夾綁在一起。

---

## 1. 先跟它聊兩句

啟動後,畫面最下面有一個輸入框。直接打字就能跟它說話,就像平常用 ChatGPT 一樣。試著問:

> 你可以幫我做什麼?

它會回你一串它能做的事。到這裡,它跟一般聊天 AI 還沒什麼兩樣 —— 真正的差別在下一步。

---

## 2. 讓它幫你動手 —— 順便認識「授權」和 `@`

Claude Code 跟一般聊天 AI 最大的不同,是它**會直接幫你動手改電腦上的檔案**。試試看,叫它建一個檔:

> 幫我建一個 notes.md,裡面寫三件我這週想學的東西

你會注意到:當它要**動到檔案**(建立、修改)時,會先跳出一個**授權提示**問你同不同意。這是 Claude Code 的安全設計 —— 它不會偷偷改你的東西,每個會動到檔案的動作都要你點頭。

> 📸 **截圖 1**:Claude Code 跳出授權提示、等你同意的畫面。

按下同意,`notes.md` 就建好了。

之後想叫它改某個**特定**的檔,可以打 `@` 再接檔名,它就會把那個檔的內容讀進來。試試:

> 把 @notes.md 的第二項改寫得更具體一點

---

## 3. CLAUDE.md:寫給它的常駐備忘錄

剛才的對話裡,它只在「這一次」記得你說的話。如果有些事你希望它**每次都記得**,可以寫進一個叫 `CLAUDE.md` 的檔:

> 幫我在這個資料夾建一個 CLAUDE.md,裡面寫:回答我的時候一律用繁體中文。

`CLAUDE.md` 的特別之處在於:**每次**你啟動 Claude Code,它都會先把這個檔讀過一遍。等於是你寫給它的「常駐備忘錄」,不用每次重講。試著之後的對話,它就會乖乖照著用繁體中文回你。

---

## 4. 它還會「主動記住」你 —— auto-memory

除了你手寫的 `CLAUDE.md`,Claude Code 還有一個更聰明的功能:它會在跟你互動的過程中,**自己判斷哪些事值得記下來**,然後存起來。這個功能叫 **auto-memory**,而且預設就開著。

試著告訴它一些關於你的事,並請它記住:

> 記住:我比較習慣用 Python,以後新專案盡量用 Python 幫我起頭。

它會把這類「關於你」的事寫進它的記憶。下次它就會參考這些,給你更貼合的協助。

---

## 🔑 關鍵時刻:咦,它怎麼「忘了」?

這裡做個小實驗,你會看到 Claude Code 記憶的真相。

先在**目前這個對話**裡,隨口跟它講一件**沒有**叫它記住的小事(例如「我今天午餐吃了拉麵」)。接著打 `/clear` 把對話清空(這等於開一個全新的對話),然後問它:

> 我剛剛說我午餐吃了什麼?

你會發現它**答不出來**。這點出 Claude Code 記憶的兩個重點:

1. **一次對話裡的內容,不會自動延續到下一次。** 每次 `/clear`(或重開)後,它對「剛才聊了什麼」是失憶的。
2. **但你寫進 `CLAUDE.md`、或叫它「記住」的事(auto-memory),會留下來。** 因為那些被寫成了**檔案**,不是只活在某一次對話裡。

換句話說:對話像一塊會被擦掉的白板,而 `CLAUDE.md` 和 auto-memory 是寫進筆記本、會留著的東西。

---

## 5. 那它到底記了什麼?用 `/memory` 看

你可能會好奇:它說它「記住」了,那些記憶到底長什麼樣、放在哪?在輸入框打:

```
/memory
```

> 📸 **截圖 2**:`/memory` 列出記憶檔、可開啟資料夾/編輯的畫面。

`/memory` 會列出目前的記憶檔,也能讓你打開那個資料夾、點開來編輯。如果你順著打開那個資料夾,會看到記憶其實就是一堆 **`.md` 純文字檔**,放在這樣的路徑底下:

- macOS:`~/.claude/projects/<專案>/memory/`
- Windows:`%USERPROFILE%\.claude\projects\<專案>\memory\`

裡面有一個 `MEMORY.md` 當索引,旁邊還有幾個主題檔。

> 📸 **截圖 3**:檔案總管裡那個 memory 資料夾,看到一堆 `.md` 檔。

---

## 撞到痛點了(下一章解決)

看到這裡你應該有個感覺:它的記憶雖然都好好存著,但要**查看**這些記憶,你只能用 `/memory` 列個清單、再一個一個用編輯器打開那些 `.md` 檔 —— **看不到全貌,也不好管理**。

如果有一個畫面,能一眼看到所有記憶、點一下就看內容、甚至直接增刪改,不是方便多了嗎?

**這就是下一章要做的事:我們會用 Claude Code,親手做一個「Claude 內視鏡」桌面 app,專門用來看它自己的記憶。**
