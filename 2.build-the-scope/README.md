# 第 2 章 — 動手做出「Claude 內視鏡」

上一章你發現:Claude 的記憶就是一堆散在資料夾裡的 `.md`,只能一個一個用編輯器開,看不到全貌。這一章我們就來解決它 —— **用 Claude Code,從零做出一個桌面 app,用畫面看它自己的記憶。**

這會是你第一個親手做出來的 app。你也會第一次體驗 **Vibe Coding**:你用「講」的描述要什麼,Claude 幫你寫 code、你跑跑看、有錯就把錯誤貼回去給它修。**你完全不需要會寫 JavaScript。**

> 做不出來、或想對答案,這章的完整成品在 [`solution/`](./solution/) —— 但強烈建議先自己跟著走一遍,真的卡住再看。

---

## 心法:先讓它規劃,別一上來就狂寫

新手最容易踩的雷,是需求還沒講清楚就讓 Claude 一股腦寫一堆 code,結果方向歪掉、改都改不完。

所以開工第一件事:切到 **Plan Mode(計畫模式)**。在輸入框按 **Shift + Tab** 切換模式,切到畫面顯示 plan mode。這個模式下,Claude **只規劃、不動手寫**,等你點頭才開工。

然後把你要的東西講清楚,例如這樣對它說:

> 我想做一個桌面 app,叫 Claude Scope。它要能讀我電腦上 `~/.claude/projects/` 各專案底下 `memory/` 資料夾裡的 `.md` 檔,左邊列出這些檔案、右邊顯示我點選那個檔的內容。用 Electron 做。先給我一份計畫,有不清楚的地方先問我,先不要動手寫。

Claude 會回你一份計畫,可能還會反問你幾個問題。看過覺得 OK,再讓它開工。

---

## 第一步:先把空殼搭起來

請 Claude 幫你把 Electron 的專案骨架建起來,它多半會用官方的 `create-electron-app` scaffold 一個能跑的空殼:

> 照計畫開始吧,先把 Electron 專案的空殼建好,然後幫我跑起來看看。

它會跑 `npm start`。**第一次會看到「Downloading Electron binary…」,要等一下**(它在下載 Electron 本體),別以為當機了。等一下,就會跳出一個空白的視窗 —— 這就是你 app 的雛形。

## 第二步:讓它讀記憶、顯示出來

空殼有了,現在請 Claude 把真正的功能填進去。**一次講一個功能,做完跑一下再做下一個**,比一次塞一大堆穩。

先做「列出記憶檔」:

> 讓左邊那一欄,列出 `~/.claude/projects/` 各專案 `memory/` 資料夾裡所有的 `.md` 檔,每個顯示檔名。

> 💡 你可能會看到 Claude 解釋:因為安全限制,畫面那一層不能直接讀檔,要透過一個叫 **preload** 的橋樑。這些它會自己處理好,你只要知道「它在用比較安全的方式讀檔」就行。

再做「點檔顯示內容」:

> 接著,我點左邊某個檔案時,右邊就顯示那個檔的完整內容。

每做完一塊,就叫它跑起來看看效果。

---

## 🔑 關鍵時刻:你的 app 真的長出來了

當你跑起來,看到左邊列出一排你的記憶檔、點一下右邊就出現內容 —— **這是你親手(用講的)做出來的第一個桌面 app。** 停下來爽一下,值得。

> 📸 **截圖**:做好的「Claude 內視鏡」—— 左邊記憶清單、右邊內容。

### 出錯了怎麼辦?(這很正常)

Vibe Coding 過程中跳出紅色錯誤訊息是**家常便飯**,不用緊張。最簡單的處理:**把錯誤訊息整段複製,貼回去給 Claude**,說「跑出這個錯,幫我修」。它修完你再跑一次,還錯就再貼一次。這個「跑 → 錯 → 貼 → 修」的循環,就是 Vibe Coding 的日常。

---

## 第三步:存個檔(Git)

功能能跑了,養成好習慣:用 Git 存一個版本,之後改壞了還能退回來。第 0 章綁好的 GitHub,這裡正好派上用場:

> 幫我把目前的進度用 git 存一個版本,訊息寫 "working memory viewer"。

---

## 成品 + 挑戰

**跟著做**(這章的基本目標):做出能「列出記憶 + 點開看內容」的內視鏡。

**進階挑戰**(行有餘力):
- 讓它能**編輯**記憶內容,改完存回檔案;
- 或能**刪除**不要的記憶;
- 或把介面**美化**一下(配色、排版隨你)。

> 編輯、刪除這些,正是把這個 app 養大的起點 —— 第 4 章我們會回來,讓它連 Skill 也一起管。
